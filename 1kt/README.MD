# Изучение основных понятий архитектуры приложений

## Зачем это необходимо

Архитектура приложений помогает понять, как устроена программа, из каких частей она состоит и как эти части взаимодействуют друг с другом.

Знание архитектуры помогает разработчикам:

- лучше понимать структуру программ
- легче поддерживать и изменять код
- создавать более удобные и масштабируемые приложения

Правильно выбранная архитектура делает программу более стабильной и понятной.

---

## Способ изучения

Для изучения архитектуры приложений были выполнены следующие действия:

- изучены теоретические материалы по архитектуре программного обеспечения
- рассмотрены примеры архитектуры веб-приложений
- изучены основные архитектурные термины
- разобрана простая схема веб-приложения

---

## Основные архитектурные термины

### Архитектура приложения
Общая структура программы и способ организации её компонентов.

### Компонент
Отдельная часть программы, которая выполняет определённую задачу.

### Слой (Layer)
Разделение приложения на уровни, где каждый уровень выполняет свою функцию.

Пример слоев:

- **Frontend** — пользовательский интерфейс  
- **Backend** — серверная часть приложения  
- **Database** — база данных для хранения информации  

### API
Интерфейс, через который разные части программы могут взаимодействовать друг с другом.

### Масштабируемость
Способность системы работать при увеличении количества пользователей или данных.

---

## Архитектура простого веб-приложения

Простое веб-приложение обычно состоит из трёх основных частей:

### 1. Frontend
Часть, которую видит пользователь.  
Отвечает за интерфейс и отображение данных.

### 2. Backend
Серверная часть, которая обрабатывает запросы пользователя и выполняет основную логику приложения.

### 3. Database
База данных, где хранится вся информация приложения.

---

## Пример схемы архитектуры
    Пользователь → Frontend → Backend → Database
    
---

## Объяснение архитектурного решения

В данной работе используется простая **трёхуровневая архитектура**.

Она разделяет приложение на три части:

- интерфейс пользователя (Frontend)
- серверную логику (Backend)
- хранение данных (Database)

Такой подход позволяет упростить разработку и поддержку приложения, так как каждая часть системы выполняет свою задачу.
